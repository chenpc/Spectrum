Spectrum Video Pipeline

一般 AVPlayer 播放流程 vs. Spectrum 自訂 Metal 渲染管線

2026-04-08

01 / STANDARD

一般播放：AVPlayer + AVPlayerLayer

Standard 最簡單的播放方式
1. 建立 AVPlayer Main
載入影片檔，AVFoundation 自動分析格式、codec
let player = AVPlayer(url: videoURL) let layer = AVPlayerLayer(player: player)
2. 等待 readyToPlay Background
背景初始化 VideoToolbox 硬體解碼器、填充 buffer
3. player.play() Main
開始播放，audio/video clock 啟動
4. 系統自動渲染 Hardware
AVPlayerLayer 自動取 frame → 顯示。App 完全不介入。
5. A/V Sync Hardware
系統自動處理音畫同步
優點
  • 程式碼約 10 行
  • 系統自動處理 HDR / DV / rotation
  • Main Thread 幾乎零負擔
  • 音畫同步由系統保證
限制
  • 無法插入自訂 shader
  • 無法控制色彩轉換
  • 無法取得 per-frame PTS
  • 無法做 gyro stabilization

03 / SPECTRUM

Spectrum 啟動階段（Load → Play）

Spectrum 7 步啟動流程
1. load(path:) Main
清理舊 player、重設狀態、記錄 loadStartTime
2. analyzeVideo(asset:) Main (async)
讀取 format descriptions → codec / bitDepth / HDR / DV / rotation
3. 自動選擇管線參數
outputPixelFormat (8/10bit × V/F) • decodeMode (601/709/2020) • colorspace (HLG/PQ/sRGB) • EDR
4. 建立 AVPlayer Main
AVPlayerItem + VideoOutput + preferredForwardBufferDuration (5s)
5. KVO: readyToPlay
等待解碼器就緒 — 確保第一幀不黑屏
6. Gyro 延遲載入（可選）Background ~300ms
waitingForGyro = true → GyroCore.start() → loadGyroCore() → 恢復播放
7. player.play() Main
readyToPlay ∧ gyro 就緒 → 開始播放 + startDisplayLink()
04 / SPECTRUM

Spectrum 渲染迴圈（每幀）

Spectrum renderFrame()
1. CVDisplayLink Tick Real-time
非阻塞 semaphore check — 前一幀未完成就跳過
guard renderSemaphore.wait(timeout: .now()) == .success
2. renderQueue.async Render
QoS = .userInteractive（不佔 real-time thread）
3. copyPixelBuffer Render
精確 PTS 用於 gyro 時間對齊
4. Gyro Matrix Render
computeMatrixAtTime(pts) → matTex
5. Metal Encode + Submit
Pass 1 + Pass 2 (見下一頁)
6. Present + Flush
cmdBuf.present(drawable) cmdBuf.commit() CVMetalTextureCacheFlush(textureCache, 0)
Frame Drop 策略
  • DispatchSemaphore(value: 1)
  • CVDisplayLink tick 時如果前一幀還在渲染 → 直接放棄此 tick
  • 保證 renderQueue 永遠只有 1 個工作
  • graceful drop，不會累積延遲
為什麼不在 CVDisplayLink thread 直接渲染？
  • CVDisplayLink 是 real-time priority
  • CoreAudio 也在 real-time 執行
  • 如果 GPU wait 卡住 → 搶佔音訊 → 爆音

07 / SPECTRUM

Dolby Vision 特例：AVPlayerLayer 直通

Dolby Vision 自動 Fallback
偵測 DV Profile
從 dvcC / dvvC atom 讀取 DV Profile & Level
啟用 AVPlayerLayer
AVPlayerLayer 疊在 CAMetalLayer 上
enableAVFLayer() // renderFrame() → avfLayerMode guard → return // currentTime 改用 4Hz periodic observer
系統處理 HDR Tone Mapping
Apple 內部自動套用 DV RPU → EDR rendering
為什麼 DV 必須用 AVPlayerLayer？
  • Dolby Vision 的 RPU（Reference Processing Unit）是 proprietary metadata
  • 只有 Apple 的 AVPlayerLayer 有 DV RPU decoder
  • copyPixelBuffer() 拿到的 pixel data 不含 RPU 處理結果
  • 直接用 Metal 渲染 DV 內容 = 丟失動態 tone mapping
  • 代價：無法套用 gyro stabilization

09 / NOTES

效能關鍵點

為什麼需要自訂管線？
  • AVPlayerLayer 無法插入 gyro warp shader
  • Sony 影片的 color matrix mislabel 無法修正
  • 需要精確 per-frame PTS 對齊 gyro 資料
  • 需支援多種 HDR 並允許手動切換
效能保護措施
  • renderSemaphore(1) — graceful frame drop
  • renderQueue .userInteractive — 不搶 CoreAudio
  • displaySyncEnabled = true — 對齊 vsync
  • CVMetalTextureCacheFlush — 回收解碼 buffer
  • framebufferOnly = true — GPU 優化
  • VideoController 4Hz poll — 最小化 Main Thread
曾導致卡頓的原因
  • Texture Cache 未 flush — buffer pool 耗盡 → 間歇掉幀
  • CVDisplayLink 直接渲染 — 搶佔 CoreAudio → 爆音
  • 無 semaphore — renderQueue 堆積 → GPU 背壓 → 延遲累積
